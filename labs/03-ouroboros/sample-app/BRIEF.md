# BRIEF — natural-priority CLI

## 한 줄 (의도적으로 모호함)

> "오늘 할 일을 자연어로 받아서 우선순위를 매겨주는 CLI를 만들어줘."

이 한 줄에는 다음 가정이 숨어 있습니다. `ooo interview`가 노출해야 합니다.

- "자연어"는 어디까지인가? 한 줄 vs 여러 줄?
- "우선순위"는 무엇을 기준으로 하는가? 마감일? 영향력? 사용자 직감?
- LLM을 호출해야 하는가, 아니면 키워드 기반 휴리스틱으로 충분한가?
- 결과는 화면 출력만? 파일 저장? 둘 다?

## 구현 제약

- Python 3.12+ 표준 라이브러리만 사용 (LLM 호출은 이번 랩 범위 밖)
- 단일 모듈 `npri.py` + 진입점 `python3 -m npri`
- 테스트는 `tests/` 디렉토리에 작성, `python3 -m unittest discover -s tests`로 실행

## Seed에 반드시 포함될 항목 (인터뷰에서 잠가야 함)

1. 입력 포맷 (한 줄 vs 여러 줄, 구분자)
2. 우선순위 알고리즘 (선택한 휴리스틱과 그 근거)
3. 출력 포맷 (텍스트 vs JSON, 결정 이유)
4. 실패 모드 (빈 입력, 인식 못 한 토큰)
5. 비목표 (LLM 호출, 원격 동기화 등 명시적으로 제외)

## 완료 정의

- Seed가 위 5개 항목을 모두 포함
- `ooo evaluate`의 3단계 게이트 통과
- `python3 -m unittest discover -s tests` 통과
